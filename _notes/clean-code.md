## Ch. 1 Clean Code

There are two parts to learning **craftsmanship**: __knowledge__ and __work__. You must gain the knowledge of principles, patterns, practices, and heuristics that a craftsman knows, and you must also grind that knowledge into your fingers, eyes, and gut by working hard and practicing.

Learning to write clean code is hard work. It requires more than just the knowledge of principles and patterns. You must sweat over it.

Be prepared to work hard while reading this book.

We have divided this book into three parts.

1. the principles, patterns, and practices of writing clean code
2. case studies
3. the payoff, single chapter containing a list of heuristics and smells gathered while creating the case studies

We will never be rid of code, because code represents the details of the requirements. At some level those details cannot be ignored or abstracted; they have to be specified. And specifying requirements in such detail that a machine can execute them is *programming*. Such a specification is *code*.

The only way to make the deadline—the only way to go fast—is to keep the code as clean as possible at all times.

Being able to recognize good art from bad does not mean that we know how to paint.

Writing clean code requires the disciplined use of a myriad little techniques applied through a painstakingly acquired sense of “cleanliness.” This “code-sense” is the key.

Clean code is pleasing to read. Reading it should make you smile the way a well-crafted music box or well-designed
car would.

Clean code does one thing well. It is no accident... Bad code tries to do too much, it has muddled intent and ambiguity of purpose. Clean code is focused.

> Clean code is simple and direct. Clean code reads like well-written prose. Clean code never obscures the designer’s intent but rather is full of crisp abstractions and straightforward lines of control.

> Clean code can be read, and enhanced by a developer other than its original author. It has unit and acceptance tests. It has meaningful names. It provides one way rather than many ways for doing one thing.

Leave the campground cleaner than you found it.
